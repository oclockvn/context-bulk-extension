## Summary
- separate SQL Server provider from core EF Core logic

## Details
- abstract provider-specific metadata via `EntityProviderBase` and resolver
- implement SQL Server metadata provider in `ContextBulkExtension.SqlServer`
- switch core projects to reference `Microsoft.EntityFrameworkCore`
- pin SQL Server provider packages for net10 and net8
- allow provider assemblies to access core internals via `InternalsVisibleTo`
